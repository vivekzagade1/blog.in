RESTful API: Internet sources say "REST is acronym for REpresentational State Transfer. It is architectural style for distributed hypermedia systems and was first presented by Roy Fielding in 2000 in his famous dissertation." In simple words "set of constraints to be used for creating Web services." Now to simplify some more. You use RESTful API to get information or "resources" from a certain server mapped to URL through hard-coded URL string or using various query-form verbal entities to get certain concentrated form of resource. Now, there are a few concepts/principle when it comes to understanding a this sophasticated piece of technology: 1. Client-Server: Breaking down storage of resources and user beneficiary we improve the portability of the user interface across multiple platforms and improve scalability by simplifying the server components. 2. Stateless: The request from client must contain all the necessary information to be able to understand the request. 3. Hierarchical layers: Structured layers of resouces inaccordance with the meta-data is positive that needs to be in our favour. 4. Uniform interfaces: Every action and every reaction in this world has a meta-meaning that has interlinked connections to the other parts of the project. 5. Cache: When a certain part of the request of a part of the response can be saved up to use for further operations is something that needs tio kept in mind.
